# 分母計算の議論 (2026-03-29)

## 現状の仕様

分母 = ワールドボーダー内(100x100)、y=-64〜63 の空気ブロック(AIR, CAVE_AIR, VOID_AIR)のうち、ブロック光レベル = 0 のもの。

以前は「全空気ブロック」を分母にして baselineLitCount で自然光源分を引いていたが、シンプルに「暗い空気ブロックのみ」に変更した。これにより baselineLitCount は不要になり削除。

## 分母が大きすぎるのでは？ という議論

実測で Total: 17242 となり、スペクテイターで見た印象より多く感じた。

### 検討した削減案と却下理由

1. **松明で照らせない空気ブロックを除外する案**
   - 各空気ブロックからマンハッタン距離14以内に固体ブロックがあるか判定
   - 計算量: 対象ブロック数 x 距離d以内のブロック数(d=14で約3000)
   - 対象が20万ブロックの場合 6億回の参照 → 重すぎて却下

2. **固体ブロックに近い空気ブロックだけ分母にする案(早期終了付き)**
   - 閾値を7にして近傍探索、固体が見つかったら打ち切り
   - 1.18以降の大洞窟(チーズ洞窟等)では数十ブロック四方が空気のみの空間が多く、大量のブロックが最大距離まで探索 → 大洞窟が多い地形では重い

3. **固体ブロック側から隣接空気ブロックを塗る案**
   - 128万ブロック1回ループで済む
   - しかし分母の定義が変わらず(暗い空気ブロックとのAND)、分母削減にならない → 却下

4. **分母を「松明を置ける場所の数」にする案**
   - 上面が固体ブロックである空気ブロック = 床の上の空気のみカウント
   - 計算はO(1)で軽い
   - しかし洞窟を探索せずにその場で松明を敷き詰めれば勝つゲームになる → ゲーム性が壊れるため却下

### 結論

17242は見た目より多く感じるが、見えていない小さな隙間や別の洞窟が地下に散在しており、実態としては妥当な数字。現状の計算方法のままとする。
